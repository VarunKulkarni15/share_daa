Architecture Analysis

Last Updated: July 2026

This document provides a deep-dive analysis of the ShareDaa architecture, dissecting its unique design system, the core local routing mechanisms, and how we solved critical network edge cases.

1. Design System & Aesthetics

ShareDaa employs a "Premium Glassmorphic" design language, prioritizing visual depth, warmth, and fluidity over flat corporate aesthetics.

  • Typography Pairings: The UI utilizes `Instrument Serif` for headers and logos, injecting an elegant, editorial feel. This is contrasted heavily by `Syne` and `Inter` for body text, providing modern geometric readability.
  • Aurora Gradient Orbits: Instead of static backgrounds, ShareDaa uses a 40-second rotating CSS linear gradient (`auroraOrbit`) that breathes life into the application frame.
  • Glassmorphism: Main panels and drop zones utilize rgba(255,255,255,0.4) backgrounds with heavy background blur. This allows the underlying Aurora gradients to bleed through the interface dynamically.

2. Core Engine (Tauri + Sockets)

Moving from Electron to Tauri was the pivotal shift for ShareDaa.

Tauri v2 Rust Vite Socket.io
  • Backend Engine: Tauri (Rust) reduces the memory footprint from ~300MB to <30MB, eliminating the bundled Chromium instance while providing native OS access.
  • The Desktop Relay: Instead of relying on slow WebRTC Data Channels and complex ICE negotiations, the Desktop app hosts its own ultra-fast WebSocket relay server (`TcpListener`) over your local LAN.
  • Data Transport: When you send a file, it is chunked and streamed directly through this raw socket at gigabit speeds, completely bypassing internet servers, STUN relays, and cloud proxies.

3. The VPN Solution

Previously, ShareDaa struggled when users were connected to VPNs (like NordVPN or ExpressVPN). VPNs often block the ZeroConf / mDNS UDP packets required for automatic device discovery.

  • The Pivot: We completely abandoned mDNS.
  • QR Code Handshake: When the Desktop app starts, it fetches its true local IPv4 address (e.g., 192.168.1.50) and generates a secure QR code.
  • Direct Connection: When you scan the QR code with your phone, the phone bypasses the VPN routing tables entirely by hitting the specific IP address and port directly, guaranteeing a 100% reliable connection every time.
